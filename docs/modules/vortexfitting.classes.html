class VelocityField

new file type reader should be added here

class vortexfitting.classes.VelocityField(file_path='/', time_step=0, mean_file_path='/', file_type='/')¶

Bases: object

Data file

Loads the input file with the NetCFD (.nc) format, a Tecplot format (.dat) or a raw / dat file; initialize the variables.

Parameters:

  • file_path (str) – file path

  • time_step (int) – current time step

  • mean_file_path (str) – in case of a mean field subtraction

  • file_type (str) – ‘PIV NetCDF (.nc)’, ‘DNS (.nc), ‘PIV Tecplot (.dat)’, ‘OpenFOAM (.raw)’, ‘HDF5 (.h5)’

  • self.x_coordinate_matrix (ndarray) – spatial mesh

  • self.y_coordinate_matrix (ndarray) – spatial mesh

  • self.z_coordinate_matrix (ndarray) – spatial mesh, optional

  • self.u_velocity_matrix (ndarray) – 1st component velocity field

  • self.v_velocity_matrix (ndarray) – 2nd component velocity field

  • self.w_velocity_matrix (ndarray) – 3rd component velocity field, optional

  • self.detection_field (ndarray) – ndarray

  • self.normalization_flag (boolean) – for normalization of the swirling field

  • self.normalization_direction (str) – ‘None’, ‘x’ or ‘y’

  • self.x_coordinate_step (float) – for homogeneous mesh, provides a unique step

  • self.y_coordinate_step (float) – for homogeneous mesh, provides a unique step

  • self.z_coordinate_step (float) – for homogeneous mesh, provides a unique step

  • self.derivative (dict) – contains ‘dudx’, ‘dudy’, ‘dvdx’, ‘dvdy’. Can be extended to the 3rd dimension

Returns:

vfield, an instance of the VelocityField class

Return type:

class VelocityField
